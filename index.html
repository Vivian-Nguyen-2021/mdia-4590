Learn about Branches

Environments

One way of organizing our code is to mimic the different environments for our project. For example, we may have one environment for developers, one for testing and collaboration, and one for a live site. In this case, we would have three branches as well. You may choose to call them development, staging, and master.

Merging

Merging is the action of including the code from one branch to another.

Beware! if there is a change on one file and the same line is changed on another, the code cannot be automatically merged. This will result in a merge conflict. You may need to resolve these manually or with the assistance of additional software. Good luck!
